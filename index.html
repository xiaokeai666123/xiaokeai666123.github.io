如何快速搭建一个独立博客

Posted on 03/18/2023 In tech

在搭建博客之前，有这么几个考量：

  • 支持 markdown 语法，便于迁移和维护；
  • 尽可能地减少 dependencies，不依赖于任何平台；
  • 只有上面两个条件满足，才能满足第三个条件：不花钱！

那么满足这几个条件的办法，就几乎只有 Github Pages + Hexo 了。

环境准备

Node js

  • 在官网安装 node js
  • 安装完成后，check node 和 npm：

Github repo

  • 在 Github create 一个新的 repo，注意：名字必须是 <username>.github.io
  • config repo 为 Github Pages

  • deploy 之后， <username>.github.io 检查是否可以正常访问

到这里，一个 Github Pages 就 setup 好了。加上一个 Jekyll theme 也是一个不错的静态博客。

Hexo

安装 Hexo CLI

  • 检查 Hexo 是否安装成功：hexo -v

创建 Hexo repo

  • 在 workspace create 一个新的 folder：hexo
  • 初始化
  • install dependencies

这样在本地就可以将 Hexo 跑起来：

访问 http://localhost:4000/ 就可以看到博客的界面了。

连接 Github，用于日后的 deploy

  • 生成 ssh 秘钥
  • 得到生成的秘钥
  • 将上面的秘钥加入到 Github
    • Github -> settings -> SSH and GPG keys
    • 新建一个 entry，随便起一个名字，比如 hexo，将上面的秘钥填进去
  • 在本地查看是否连接到 Github

配置 Hexo deploy

  • 在 _config.yml config Github 的 repo 地址
  • 在根目录安装一个 deploy 插件
  • 这样每次就可以把新的文章 deploy 到远程的 Github

NEXT theme

NEXT official doc

NEXT 可以支持更多的 features。

安装 NEXT

  • manual 推荐 npm，我们也可以用 git
  • 记得在 _config.yml 把 theme 改成 next

配置 NEXT

  • 在 theme/next 下的 _config.yml 可以对 NEXT 进行定制化的配置，这里不展开讨论。
